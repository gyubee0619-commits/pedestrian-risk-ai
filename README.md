# Pedestrian Risk AI

## 프로젝트 소개
pedestrian Risk AI는 이미지에서 자동차, 오토바이, 자전거 등의 위험 객체를 인식하고, 이를 기반으로 보행자 관점의 위험 상황을 분석하기 위한 프로젝트입니다.
본 프로젝트는 YOLO 기반 객체 검출 기술을 활용하여 객체 인식 결과를 바탕으로 이후 위험도 판단 로직 및 화면 시각화 기능을 연결 하는 것을 목표로 합니다. 

## 프로젝트 목표
- 보행자 위험 상황과 관련된 객체를 탐지한다.
- 탐지된 객체의 종류, 위치, 크기 정보를 추출한다.
- 추출된 정보를 기반으로 위험도 판단 로직을 구현한다.
- 객체 인식 결과와 위험도 결과를 사용자가 직관적으로 이해할 수 있는 UI로 시각화한다.

## 팀 역할 분담
1) 객체 인식 파트
- 이미지/영상에서 객체 탐지
- 객체의 종류(class), 위치(bounding box), 크기(size) 추출
- 학습 데이터셋 정리 및 모델 적용
2) 위험도 계산 파트
- 객체 인식 결과를 입력으로 받아 위험도 판단 규칙 구현
- 객체 간 거리, 화면 내 위치, 객체 크기 변화 등을 기반으로 위험 상황 계산
3) UI 시각화 파트
- 객체 인식 결과 및 위험도 결과를 사용자에게 시각적으로 표시
- 경고 문구, 위험도 단계, 박스/레이블 등 시각화 요소 구현

## 현재 구현 범위
- YOLO 기반 객체 인식
- 입력 이미지에서 객체 탐지
- 탐지된 객체의 종류(class), 위치(bounding box), 크기(size) 확인 가능
- 객체의 종류/위치 관계를 기반으로 한 위험도 계산
- 위험 상황 경고 로직
- 결과 시각화 UI

## 각 폴더 설명
1. dataset/

객체 인식 학습 및 검증에 사용되는 데이터셋이 저장된 폴더입니다.
train, valid, test, images, labels, data.yaml 등의 구조를 포함합니다.

2. models/

학습된 모델 가중치 또는 모델 관련 파일을 저장하는 폴더입니다.
추후 best model, custom weight 파일 등을 이 폴더에 저장할 수 있습니다.

3. outputs/

객체 인식 결과 이미지, 추론 결과, 시각화 결과 등 실행 결과를 저장하는 폴더입니다.

4. src/

객체 인식, 위험도 계산, UI 시각화 등 주요 코드가 위치하는 폴더입니다.
향후 기능별로 파일을 분리하여 관리할 수 있습니다.

detect_objects.py : 객체 검출 및 정보 추출 코드
risk_analysis.py : 위험도 계산 및 위험 등급 분류 코드
ui.py : 시각화/UI 코드

5. test_images/

객체 인식 테스트에 사용하는 입력 이미지들을 저장하는 폴더입니다.

6. train/

학습 과정에서 사용된 파일 또는 학습 관련 리소스를 저장하는 폴더입니다.


## 실행방법
1) 필요한 라이브러리 설치

프로젝트 루트에서 아래 명령어를 실행한다.

pip install -r requirements.txt

2) 객체 인식 코드 실행

실행 파일 위치 및 파일명은 실제 소스 코드 구조에 맞게 수정한다.

예시:

python src/object_detect.py


## 입력 및 출력
1) 입력
- 테스트 이미지
- 학습된 모델 파일(best.pt)
2) 출력
- 탐지된 객체의 클래스
- 바운딩 박스 좌표
- 객체 크기 정보
- 위험도 표시
  
## 시스템 구성 흐름
본 프로젝트의 전체 흐름은 다음과 같습니다.

1단계: 객체 인식

입력 이미지 또는 영상 프레임을 받아 객체를 탐지한다.
이 단계에서는 다음 정보를 얻는다.

- 객체 종류
- 바운딩 박스 좌표
- 객체 크기

2단계: 위험도 계산

객체 인식 결과를 기반으로 위험도를 계산한다.
예를 들어 다음과 같은 요소를 활용할 수 있다.

- 객체 종류
  - 자동차: 20점
  - 오토바이: 15점
  - 자전거: 10점

- 객체의 위치
  - 중앙 영역: 40점
  - 중간 영역: 25점
  - 가장자리 영역: 10점

- 객체 크기
  - 이미지 면적의 15% 이상: 40점
  - 5~15%: 25점
  - 5% 미만: 10점

최종 위험도는 각 점수를 합산하여 계산한다.

  
3단계: UI 시각화

계산된 위험도 결과를 화면에 표시한다.
예:

- 객체 바운딩 박스 표시
- 객체 이름 및 위험도 표시
- 위험 상황 발생 시 경고 메시지 출력
- 위험도 단계별 시각적 강조

## 데이터 흐름 예시
프로젝트 내 모듈 간 데이터 흐름은 아래와 같다.

```text
입력 이미지 / 영상
        ↓
[객체 인식 모듈]
        ↓
객체 종류 / 좌표 / 크기 / confidence
        ↓
[위험도 계산 모듈]
        ↓
위험도 결과 (예: 안전 / 주의 / 위험)
        ↓
[UI 시각화 모듈]
        ↓
최종 결과 화면 출력
```

## 향후 계획
본 프로젝트는 현재 객체 인식 파트를 중심으로 구성되어 있으며,
이후 다음 기능을 추가 및 고도화핳 예정이다.
-이미지뿐만 아니라 영상/실시간 입력에 대한 확장
-객체의 움직임에 따른 위험도 기준 세분화 
-위험도 결과 로그 저장 및 분석 기능추가

## 기대효과
본 시스템은 보행자 시점에서 차량, 오토바이, 자전거가 등장하는 상황에서
위험 가능성을 자동으로 분석하고 시각적으로 보여줌으로써, 보행자 안전 관려 AI시스템의 기초 구조를 구현하는 데 목적이 있다.
