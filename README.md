# Pedestrian Risk AI

## 프로젝트 소개
pedestrian Risk AI는 이미지에서 자동차, 오토바이, 자전거 등의 위험 객체를 인식하고, 이를 기반으로 보행자 관점의 위험 상황을 분석하기 위한 프로젝트입니다.
본 프로젝트는 YOLO 기반 객체 검출 기술을 활용하여 객체 인식 결과를 바탕으로 이후 위험도 판단 로직 및 화면 시각화 기능을 연결 하는 것을 목표로 합니다. 

## 프로젝트 목표
- 보행자 위험 상황과 관련된 객체를 탐지한다.
- 탐지된 객체의 종류, 위치, 크기 정보를 추출한다.
- 추출된 정보를 기반으로 위험도 판단 로직을 구현한다.
- 객체 인식 결과와 위험도 결과를 사용자가 직관적으로 이해할 수 있는 UI로 시각화한다.

## 팀 역할 분담
*이미지 수집은 균등한 분량으로 나눠 수집하였음.
1) 객체 인식 파트 (이규비)
- Roboflow 이미지 라벨링
- 이미지에서 객체 탐지
- 객체의 종류(class), 위치(bounding box), 크기(size) 추출
- 학습 데이터셋 정리 및 모델 적용
2) 위험도 계산 파트 (우은민)
- 객체 인식 결과를 입력으로 받아 위험도 판단 규칙 구현
- 객체 종류, 위치, 크기를 기반으로 위험 상황 계산
- 문서화 담당
3) UI 시각화 파트 (김채윤)
- Roboflow 추가 이미지 라벨링
- 객체 인식 결과 및 위험도 결과를 사용자에게 시각적으로 표시
- 경고 문구, 위험도 단계, 박스/레이블 등 시각화 요소 구현
  
## 개발 환경 및 사용 기술

본 프로젝트는 보행자 시점 이미지에서 이동체 객체를 검출하기 위한 AI 시스템 구현을 목표로 하였으며, Python 기반 환경에서 개발하였다. 객체 검출 모델 학습에는 **Ultralytics YOLOv8**를 사용하였고, 데이터셋 구축을 위해 **Roboflow**를 활용하였다. 또한 검출 결과를 이미지 상에 시각화하고 객체의 위치 및 크기 정보를 출력하기 위해 **OpenCV**를 사용하였다.

### 개발 환경
- **OS**: Windows
- **Language**: Python 3.x
- **IDE / Editor**: Visual Studio Code
- **실행 환경**: CMD / PowerShell

### 사용 기술 및 도구
- **YOLOv8 (Ultralytics)**  
  객체 검출 모델 학습 및 테스트에 사용하였다. 직접 구축한 데이터셋을 기반으로 fine-tuning하여 자동차, 오토바이, 자전거 객체를 검출하도록 학습하였다.

- **Roboflow**  
  직접 수집한 이미지 데이터에 대해 객체 라벨링을 수행하고, 이를 YOLO 학습 형식으로 export하는 데 사용하였다.

- **OpenCV**  
  테스트 이미지에서 검출된 객체의 bounding box를 시각화하고, 객체 종류·위치·크기 정보를 출력하는 기능 구현에 사용하였다.

- **GitHub**  
  프로젝트 코드, 데이터 구조, 모델 결과 파일을 공유하고 버전을 관리하는 협업 도구로 활용하였다.



## 데이터셋 설명

본 프로젝트의 데이터셋은 **보행자 시점에서 촬영한 도로 및 골목길 이미지**를 기반으로 직접 구축하였다. 보행자 우선 도로 또는 보행자 관점에서 주변 이동체를 확인할 수 있는 장면을 중심으로 이미지를 수집하였으며, 프로젝트 목적에 맞는 객체 검출 데이터셋을 구성하고자 하였다.

수집한 이미지는 **Roboflow 웹사이트**를 이용하여 라벨링하였으며, 학습 대상 클래스는 다음 3가지로 설정하였다.

- **car**
- **motorcycle**
- **bicycle**

각 이미지에서 해당 객체가 존재하는 영역에 대해 바운딩 박스(Bounding Box) 형태로 라벨링을 수행하였고, 이후 Roboflow에서 **YOLO 학습 형식**으로 export하여 모델 학습에 활용하였다. 전체 데이터셋은 약 **600장 규모**이며, 학습용(train) 데이터와 검증용(valid) 데이터로 분할하여 사용하였다.

데이터셋 폴더는 YOLO 형식에 맞추어 다음과 같이 구성하였다.

- `dataset/train/images` : 학습용 원본 이미지
- `dataset/train/labels` : 학습용 라벨 파일
- `dataset/valid/images` : 검증용 원본 이미지
- `dataset/valid/labels` : 검증용 라벨 파일

라벨 파일은 YOLO 형식의 `.txt` 파일로 저장되며, 각 파일에는 이미지 내 객체의 **클래스 번호와 정규화된 bounding box 좌표 정보**가 포함된다.


## 현재 구현 범위
- YOLO 기반 객체 인식
- 입력 이미지에서 객체 탐지
- 탐지된 객체의 종류(class), 위치(bounding box), 크기(size) 확인 가능
- 객체의 종류/위치/크기를 기반으로 한 위험도 계산
- 위험 상황 경고 로직
- 결과 시각화 UI

## 각 폴더 설명
1. dataset/

객체 인식 학습 및 검증에 사용되는 데이터셋이 저장된 폴더입니다.
train, valid, test, images, labels, data.yaml 등의 구조를 포함합니다.

2. dataset/train/
   
모델 학습에 사용되는 데이터가 저장된 폴더이다.
학습 이미지와 해당 이미지에 대한 라벨 정보가 포함되어 있으며, 모델이 객체의 특징을 학습하는 데 사용된다.

3. dataset/valid/

학습된 모델의 성능을 검증하기 위한 데이터가 저장된 폴더이다.
모델이 학습 과정에서 보지 않은 데이터를 대상으로 예측 성능을 평가하는 데 사용된다.

4. dataset/train/images/

학습용 원본 이미지 파일들이 저장된 폴더이다.
보행자 시점에서 직접 수집한 도로 및 이동체 관련 이미지들이 포함된다.
   
5. dataset/train/labels/

학습용 이미지에 대한 객체 라벨 정보가 저장된 폴더이다.
각 txt 파일에는 이미지 내 객체의 클래스 번호와 바운딩 박스 좌표 정보가 YOLO 형식으로 기록되어 있다.

6. dataset/valid/images/

검증용 원본 이미지 파일들이 저장된 폴더이다.
모델이 학습 과정에서 사용하지 않은 이미지로 구성되며, 일반화 성능을 확인하는 데 사용된다.

7. dataset/valid/labels/

검증용 이미지에 대한 객체 라벨 정보가 저장된 폴더이다.
검증 이미지에 포함된 객체의 클래스와 위치 정보가 YOLO 형식으로 저장되어 있으며, 모델 성능 평가에 활용된다.

8. models/

학습된 모델 가중치 또는 모델 관련 파일을 저장하는 폴더입니다.
추후 best model, custom weight 파일 등을 이 폴더에 저장할 수 있습니다.

9. outputs/

객체 인식 결과 이미지, 추론 결과, 시각화 결과 등 실행 결과를 저장하는 폴더입니다.

10. src/

객체 인식, 위험도 계산, UI 시각화 등 주요 코드가 위치하는 폴더입니다.
향후 기능별로 파일을 분리하여 관리할 수 있습니다.

detect_objects.py : 객체 검출 및 정보 추출 코드
risk_analysis.py : 위험도 계산 및 위험 등급 분류 코드
ui.py : UI 시각화 코드

10. test_images/

객체 인식 테스트에 사용하는 입력 이미지들을 저장하는 폴더입니다.

11. train/

학습 과정에서 사용된 파일 또는 학습 관련 리소스를 저장하는 폴더입니다.

12. requirements.txt/
해당 프로젝트 실행 시 설치가 필요한 라이브러리를 안내한 파일입니다.

13. yolo8n.pt
YOLOv8의 기본 사전학습(pretrained) 모델 파일이다.
COCO 데이터셋 등 대규모 데이터로 미리 학습된 가중치를 포함하고 있으며, 본 프로젝트에서는 이 모델을 기반으로 직접 구축한 데이터셋을 추가 학습(fine-tuning)하여 사용하였다.

## 모델 학습 방법

본 프로젝트에서는 **Ultralytics YOLOv8n** 모델을 기반으로 객체 검출 학습을 수행하였다. `yolov8n.pt`는 사전학습된(pretrained) 기본 모델 파일이며, 이를 초기 가중치로 사용하여 직접 구축한 보행자 시점 데이터셋에 대해 추가 학습(fine-tuning)을 진행하였다.

라벨링이 완료된 데이터셋은 Roboflow에서 **YOLO 형식으로 export**한 뒤 프로젝트 폴더에 저장하였으며, 데이터 경로와 클래스 정보는 `data.yaml` 파일을 통해 설정하였다. 이후 아래 명령어를 사용하여 모델 학습을 수행하였다.

```bash
yolo detect train data=data.yaml model=yolov8n.pt epochs=30
```

## 시스템 구성 흐름
본 프로젝트의 전체 흐름은 다음과 같습니다.

1단계: 객체 인식

입력 이미지 또는 영상 프레임을 받아 객체를 탐지한다.
이 단계에서는 다음 정보를 얻는다.

- 객체 종류
- 바운딩 박스 좌표
- 객체 크기

2단계: 위험도 계산

객체 인식 결과를 기반으로 위험도를 계산한다.
예를 들어 다음과 같은 요소를 활용할 수 있다.

- 객체 종류
  - 자동차: 20점
  - 오토바이: 15점
  - 자전거: 10점

- 객체의 위치
  - 중앙 영역: 40점
  - 중간 영역: 25점
  - 가장자리 영역: 10점

- 객체 크기
  - 이미지 면적의 15% 이상: 40점
  - 5~15%: 25점
  - 5% 미만: 10점

최종 위험도는 각 점수를 합산하여 계산한다.

  
3단계: UI 시각화

계산된 위험도 결과를 화면에 표시한다.
예:

- 객체 바운딩 박스 표시
- 객체 이름 및 위험도 표시
- 위험 상황 발생 시 경고 메시지 출력
- 위험도 단계별 시각적 강조

## 데이터 흐름 예시
프로젝트 내 모듈 간 데이터 흐름은 아래와 같다.

```text
입력 이미지 / 영상
        ↓
[객체 인식 모듈]
        ↓

        ↓

        ↓
최종 결과 화면 출력
```
